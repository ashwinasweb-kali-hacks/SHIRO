Deployment Guide

Can I host Shiro AI
on Netlify?

Yes, but there are some important details to know about running local AI in a cloud environment.

Netlify Static Hosting vs. Local AI

Netlify is great for the Shiro interface, but your AI models must still run on a PC with a GPU.

❌ The Disadvantages
  • Netlify cannot run Ollama or Python (No Chat/Image generation directly on their servers).
  • Browsers block connections from HTTPS (Netlify) to HTTP (Localhost) — this is "Mixed Content".
  • You cannot use the shiro:// launch protocol from a public Netlify website for security.
✅ How to Fix It
  • Deploy the Frontend to Netlify so your site is public.
  • Install cloudflared or ngrok on your PC to create an HTTPS Tunnel to Ollama.
  • Paste your Tunnel URL into Shiro's Settings > Ollama Host.
  • Now you can chat with your HOME computer from ANYWHERE in the world!

Netlify Deployment — What Works & What Doesn't

Shiro AI is a local-first app. Here's what happens when you host the frontend on Netlify:

❌ Won't Work on Netlify AI Chat needs Ollama running on your computer.
Image/Video generation needs the Python backend running locally.
Netlify is static hosting — it can't run Python or Ollama.
✅ Works on Netlify ✓ Home page & landing page
✓ Login / register page
✓ QR Code Studio (uses external free API)
✓ Chat Studio (if you set up ngrok/Cloudflare tunnel pointing to your local Ollama)
✓ Image/Video (if you deploy backend to Render.com or Railway)

💡 Best setup for Netlify: Deploy frontend to Netlify → run cloudflared tunnel on your PC → paste the tunnel URL in Shiro's Settings → Ollama Host field. Chat will work instantly. For image/video, deploy the Backend/main.py to Render.com (free tier).
